## 1) Діаграма прецедентів  

<center>
    
@startuml
    
    actor Гість
    actor Користувач 
    actor Спеціаліст
            
    usecase "Авторизуватись" as A 
    usecase "Зареєструватись" as R
            
    usecase "Управління опитуванням" as M
            
    usecase "Пройти опитування" as E
    usecase "Знайти опитування" as F 
            
    Користувач -d-|> Гість 
    Спеціаліст -d-|> Гість
            
    Гість -l-> A 
    Гість -r-> R
            
    Спеціаліст -u-> M
            
    Користувач -u-> E
    Користувач -l-> F
    
@enduml 
**Рис. 1** Ієрархія прецедентів та їх основні можливості
</center>
    
Існує 3 типи користувачів: **гість**, **користувач** та **спеціаліст**;  
**Гість** — неавторизований користвувач;  
**Користувач** — авторизований користувач, який може проходити та шукати опитування;  
**Спеціаліст** — авторизований користувач, який може працювати з опитуваннями та з їхніми результатами;
